# Performing Packet Capture and Traffic Analysis

## Overview
This project focused on capturing and analyzing network traffic to understand how data moves across a network and how security issues can be identified through packet inspection. The work emphasized monitoring protocols, reviewing packet details, and recognizing the security impact of plaintext versus encrypted communications. It also introduced wireless traffic analysis and basic malicious traffic detection. :contentReference[oaicite:0]{index=0}

## What I Worked On
In this project, I used Wireshark to capture and inspect network packets across multiple scenarios, including FTP, SFTP, ICMP, HTTP, and SSH traffic. I reviewed packet details such as IP addresses, ports, payload data, login-related information, passive FTP connections, and encrypted SSH session details. This helped me compare unencrypted and encrypted traffic and better understand how protocol behavior appears in packet captures. :contentReference[oaicite:1]{index=1}

I also worked through wireless traffic analysis by reviewing SSID, channel information, ICMP and HTTP packet details, and key elements of the four-way handshake after updating WLAN security settings. In the challenge section, I analyzed malicious traffic by reviewing deauthentication activity and related packets, which reinforced how packet captures can be used to investigate abnormal or hostile network behavior. :contentReference[oaicite:2]{index=2}

## Key Takeaways
This project strengthened my understanding of how packet analysis supports troubleshooting, security monitoring, and incident investigation. It also reinforced the importance of encryption, traffic visibility, and protocol awareness when evaluating network behavior. From an IT and security support perspective, the project highlighted how tools like Wireshark and tcpdump can help identify misconfigurations, insecure transmissions, and suspicious activity before they become larger issues. :contentReference[oaicite:3]{index=3}
